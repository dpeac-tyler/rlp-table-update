Create Questions

Use the question builder below to create your questions. Choose the type(s) of questions you want to include on your application and then provide the additional information that is necessary for each question. Keep repeating the process until you have created all the necessary questions.

For each question, you have the following options:

Check “Required” if the user must answer the question. Table and Addresses/Repeatable Fields have different settings to address “Required”

Check “Instructions” if you want to include more information that may be helpful for the user.

Required - indicates a required field.

Question Type

Note: Questions are optional by default. Add y in the y/n field to make a question required. Empty or n = optional.

Acknowledgement

ack | statement | y/n | instructions

Can be required or optional. Defaults to optional if y/n field is omitted.

Addresses / Repeatable Fields

addr | question | y/n | repeat(y/n) | instructions

repeat(y) allows multiple addresses, repeat(n) for single address. Repeat field is optional, defaults to repeat(n).

Date / Date Range

dt | question | y/n | type(single/range) | instructions

Type field is optional. type(single) = single date, type(range) = date range. Defaults to single.

Free Form

ff | question | y/n | instructions | format(an/num) | length(s/m/l/xl)

Format: an = alphanumeric, num = numeric | Length: s = short, m = medium, l = large, xl = extra large

Multiple Choice

mc | question | y/n | choices(option1,option2,option3) | instructions

Minimum 2 choices required. Use choices() wrapper.

Multi-Select

ms | question | y/n | choices(a,b,c,...) | min/max(x/y) | instructions

Multiple checkboxes. Min/max field is optional (e.g., min/max(1/3) requires 1-3 selections). Minimum 2 choices required.

Table

tbl | question | y/n | columns(Name,Age,Relationship) | rows(3) | instructions

Minimum 2 columns required. Rows field is optional. Use rows(n) where n >= 1.

Table (Row Labels)

tblr | question | y/n | rows(Label1,Label2,Label3) | columns(Header2,Header3) | instructions

Structured data-entry grid. Row labels are agency-defined (first column), columns are input fields. Minimum 1 row label and 1 column required.

Yes / No

yn | question | y/n | labels(option1,option2) | instructions

Labels field is optional. Defaults to Yes/No if omitted.
